# 原理

  • 先执行一个宏任务、再执行所有的微任务（反复这个循环）

# 异步任务 (opens new window)

  • 宏任务（Macro-Task）：参与事件循环的异步任务（别的线程里面完成的 -- 怎么理解？）
    • eg: I/O、request-Animation-Frame、set-Timeout、set-Interval、set-Immediate
  • 微任务（Micro-Task）：不参与事件循环的"异步" (opens new window)(假异步)任务 -（同一个事件循环中，比宏任务优先级高、主线程任务完成后立即执行， 没有在别的线程里完成）
    • eg: process_next-Tick (node.js)、Mutation-Observer (opens new window)、Promise

# 拓展

  • Chrome事件循环（Event-Loop)
  • 事件驱动（Event Driven) - (发布订阅模式/观察者模式 (opens new window))
  • 消息循环（Message-Loop）
  • 运行循环（Run-Loop）
  • Mutation Events (opens new window) - 是在 DOM3 (opens new window)中定义
  • Mutation-Observer (opens new window) - 是在 DOM4 中定义
