    # 概念

    PostCSS 接收一个 CSS 文件并提供了一个 API 来分析、修改它的规则（通过把 CSS 规则转换成一个抽象语法树 (opens new window)的方式）。在这之后，这个 API 便可被许多插件 (opens new window)利用来做有用的事情

    • 具体语法树（分析树 - Parse Tree）
    • 抽象语法树（AST - Abstract Async Tree）

    语法分析器 (opens new window)创建出分析树，然后从分析树生成AST。一旦AST被创建出来，在后续的处理过程中，比如语义分析 (opens new window)阶段，会添加一些信息

    一个现代编译器的主要工作流程：

    源代码 (opens new window)（source code）→ 预处理器 (opens new window)（preprocessor）→ 编译器 (opens new window)（compiler）→ 汇编程序 (opens new window)（assembler）→ 目标代码 (opens new window)（object code）→ 链接器 (opens new window)（linker）→ 可执行文件 (opens new window)（executables）
